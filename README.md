# 🎨 TynanArtSimulator / 泰南艺术生成器

一个受 **RimWorld 高品质物品艺术描述机制** 启发的中文文本生成器。  
项目当前以 **纯静态 Web 版** 为主，适合直接部署到 **GitHub Pages**。

---

## ✨ 项目特色

- 🧠 **机制导向生成**：不是普通随机文案，而是按“开头句 / 主画面 / 细节 / 风格 / 背景句”分层组合。
- 🈶 **中文本地化输出**：围绕中文表达习惯重写了句式池与描述节奏。
- 🧩 **可解释**：内置“生成机制”页面，可直接查看事件槽位、句式池和生成逻辑。
- 🛠️ **可自定义**：支持创建自定义事件模板、自定义角色与对象槽位、句式变量与元字段。
- 📱 **移动端友好**：横屏 / 竖屏均可使用，竖屏下会自动转为更适合触控的纵向布局。
- 🌗 **主题切换**：支持日间 / 夜间模式切换。
- 💾 **本地保存**：人物预设、自定义事件、主题偏好会保存在浏览器本地。
- 🚀 **静态部署友好**：无需 Node 构建流程，直接放到 GitHub Pages 即可运行。

---

## 🧱 主要功能

### 1. 文本生成器
- 选择事件类型
- 选择载体类型（雕塑 / 家具 / 近战武器 / 远程武器 / 石棺饰板）
- 选择简易模板 / 完整模板
- 一次生成 1～5 条候选文本
- 支持复制单条或复制全部结果

### 2. 生成机制页面
- 浏览系统事件与自定义事件
- 查看每个事件所需槽位
- 查看可能出现的句式
- 直观理解艺术文本如何由多层结构组合而成

### 3. 自定义事件模板
- 自定义事件标题
- 自定义必填 / 可选角色槽位
- 自定义必填 / 可选对象槽位
- 自定义事件叙述、主画面、细节句、风格句、场景短语
- 实时预览生成结果与可用变量

### 4. 人物预设
- 保存常用角色姓名 / 称呼 / 代词
- 在生成器中优先提供建议

## 🚀 使用方法

### 方式一：直接打开

如果只是本地快速使用，可以直接打开：

```text
Online/index.html
```

### 方式二：本地启动静态服务

建议在项目根目录执行：

```bash
python -m http.server 8000
```

然后访问：

```text
http://127.0.0.1:8000/Online/index.html
```

### 方式三：部署到 GitHub Pages

本项目当前是纯静态结构，不依赖 Node 构建。  
只需将 `Online/` 中的静态资源发布到 GitHub Pages 即可。

推荐步骤：
1. 将 `Online/` 作为站点发布目录
2. 确保 `index.html`、`styles.css`、`content-data.js`、`app.js` 一并上传
3. 开启 GitHub Pages

## 🧭 典型使用流程

### 生成艺术文本
1. 进入“生成器”
2. 选择事件类型
3. 填写人物 / 对象 / 时间
4. 点击“生成”
5. 从候选结果中挑选最满意的一条

### 创建自定义事件
1. 进入“自定义事件”
2. 填写事件标题
3. 添加角色与对象槽位
4. 补充句式模板
5. 点击“保存/更新事件”
6. 回到“生成器”即可直接使用

## 🧠 关于句式扩展

项目中的艺术文本句式，参考了 RimWorld 艺术描述相关规则，并结合中文表达进行了本地化改写。  
在近期更新中，基于 MCP 读取到的源码 / RulePackDef 结构，对以下部分进行了扩展：

- 载体开头句
- 背景句
- 标题句
- 默认细节句 / 默认风格句
- 每个事件模板下的：
  - `event_nouns`
  - `main_images`
  - `detail_sentences`
  - `style_sentences`
  - `scene_phrases`

目前大多数事件模板的句式池，已经从原先的 **3 条左右** 扩展到了 **10 条**。

## 📌 注意事项

- 本项目更偏向 **风格模拟 / 机制复现**，不是 1:1 原版逐字复刻。
- 自定义事件过多、内容过长时，浏览器本地存储会接近容量上限。
- 不同浏览器对本地文件与存储权限的处理略有差异，建议优先使用静态服务器或 GitHub Pages 访问。
