# 🎨 TynanArtSimulator / 泰南艺术生成器

一个受 **RimWorld 高品质物品艺术描述机制** 启发的中文文本生成器。  
它可以用来快速生成类似游戏中“高品质物品附带艺术描述”的中文文本，也可以让你自己定义事件模板、角色槽位和句式池。

---

## ✨ 这个项目能做什么

- 生成 RimWorld 风格的中文艺术文本
- 一次生成多条候选结果进行挑选
- 支持按需显示英文参考文本
- 查看文本是如何由不同句式组合出来的
- 创建自己的事件模板与句式池
- 保存人物预设与自定义事件，方便重复使用
- 在电脑和手机上都能较舒服地进行操作
- 以更接近游戏原规则的方式组合句式与参考英文结构
- 提供 `rules.md` 方便查看当前接入的 grammar 规则说明

## 🧩 主要功能

### 1. 生成器
- 选择事件类型
- 选择载体类型（雕塑 / 家具 / 近战武器 / 远程武器 / 棺材）
- 选择简易模板或完整模板
- 一次生成 1～5 条候选文本
- 可开关显示英文参考文本
- 支持复制单条结果或复制全部结果

### 2. 生成机制
- 查看每种事件需要哪些输入
- 查看可能出现的句式
- 理解文本结构：开头句、主画面、细节、风格、背景句

### 3. 自定义事件
- 新建自己的事件模板
- 配置必填 / 可选角色
- 配置必填 / 可选对象
- 编写主画面、细节句、风格句、背景句、场景短语
- 通过实时预览检查效果

### 4. 人物预设
- 保存常用角色姓名 / 称呼 / 代词
- 在生成器中优先作为建议项出现

## 🧠 关于句式库

项目中的句式并不是普通随机文本，而是按 RimWorld 艺术描述机制的思路进行组织。  
当前 Online 分支已经改为 **“官方 RulePack / TaleDef grammar + 槽位替换”** 的主生成路径。  
也就是说，内置事件优先直接使用：

- `ChineseSimplified` 中的官方简中 `TaleDef.rulePack.rulesStrings`
- `RulePacks_Art_Descriptions*.xml` 中的官方简中艺术规则
- MCP 确认过的 `CompArt -> TaleTextGenerator -> RulePackDef` 真实调用链

因此，现在的核心不再是“手写几句像游戏的话”，而是尽量按游戏原本的规则结构来展开文本。

扩展过的内容包括：
- 载体开头句
- 背景句
- 标题句
- 默认细节句 / 默认风格句
- 各事件模板中的主画面、背景叙述、细节句、风格句、场景短语

同时也针对旧版本中存在的错译、漏填 token、错误 role symbol、把 `HasTale` / `Taleless` 混用等问题做了集中修复；并追加了可开关显示的英文参考，方便对照游戏原本的艺术描述组织方式。

项目现在也会参考游戏内简体中文规则资源中的部分艺术文本句式与 TaleDef 事件描述，以尽量让中文表达更接近游戏版本本身。  
当前生成器的核心也已经逐步转向“规则池 + 槽位替换”的模式，而不是只依赖手写句式模板。  
部分常见译名（如品质、武器、材料、载体名称）也已经按游戏简体中文资源进行统一修正。
常用预设词条（如物品名、建筑名、研究项目名、游戏设施名、疾病名、技能名）也会尽量直接使用官方简中 label，而不是项目早期的自拟译名。

目前系统事件中的很多内容，已经直接参考游戏内 `TaleDef` 与 `RulePackDef` 的文本结构，而不再完全依赖项目早期手写的事件模板。  
如需查看这套结构当前到底接了哪些规则，可直接阅读仓库里的 `rules.md`。

## 🌐 使用方式

项目主要面向 GitHub Pages 访问。  
使用时，直接通过项目提供的网页入口打开即可，无需额外安装或部署本地环境。

---

## 📱 使用建议

- 想快速写文本：使用 **简易模板**
- 想细调风格：使用 **完整模板**
- 想长期积累自己的事件：使用 **自定义事件**
- 想在手机上操作：直接竖屏使用即可，界面会自动转为更适合纵向阅读的布局

---

## 💾 本地保存说明

项目会把以下内容保存在浏览器本地：
- 人物预设
- 自定义事件
- 主题偏好

如果你更换浏览器或清理浏览器本地数据，这些内容可能会消失。
